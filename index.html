Methodology

The methodology for music genre classification involved the following key steps:

  1. Data Collection: The dataset was obtained from the GTZAN Dataset, a widely used collection of audio files spanning various music genres.
  2. Feature Extraction: Relevant features were extracted from the audio files using the librosa library in Python. These features included tempo, spectral centroid, zero-crossing rate, and Mel-frequency cepstral coefficients (MFCCs).
  3. Data Preprocessing: Before training the machine learning models, the extracted features underwent preprocessing. This process involved scaling the features to a standard range and splitting the dataset into training and testing sets.
  4. Model Selection:Different machine learning algorithms, including Decision Trees, AdaBoost, Logistic Regression, Naive Bayes, and K-Nearest Neighbors (KNN), were experimented with to determine the best performer for music genre classification.
  5. Model Evaluation: The performance of each model was evaluated using techniques such as cross-validation and grid search to tune hyperparameters. Metrics such as accuracy, precision, recall, and F1-score were assessed to gauge the effectiveness of each model.

Following this methodology, a series of experiments were conducted to build and evaluate machine learning models capable of accurately classifying music genres based on audio features.

Music Genre Classifiers

In the pursuit of music genre classification, a diverse array of machine learning methods were explored, each showcasing unique characteristics and performance. Below provides a concise overview of the methodologies employed:

  1. Decision Tree Classifier: The initial approach involved the utilization of Decision Trees, which segment the feature space into hierarchical decision rules. Through the application of Grid-SearchCV, hyperparameters were fine-tuned to optimize classification accuracy. Despite its simplicity, Decision Trees served as a robust baseline for genre classification tasks.
  2. AdaBoost Classifier: Recognizing the limitations of Decision Trees, an ensemble learning technique known as AdaBoost was adopted. AdaBoost combines multiple weak classifiers to form a stronger ensemble model. Various hyperparameter combinations, such as the number of estimators and learning rate, were explored to enhance classification performance.
  3. Gaussian Naive Bayes Classifier: Another explored method was the Gaussian Naive Bayes classifier, a probabilistic model that assumes feature independence given the class label. Despite its simplicity, Naive Bayes demonstrated rapid training times and competitive performance in genre classification tasks.
  4. Logistic Regression: Lastly, Logistic Regression, a linear model well-suited for multiclass classification, was investigated. Parameter tuning, including adjustments to regularization strength (C) and penalty type, aimed to identify the optimal configuration for precise genre classification.
  5. KNN (K-Nearest Neighbors) Classifier: Additionally, the KNN classifier was explored, which is a non-parametric method used for classification. It classifies objects based on the majority class among their k nearest neighbors in the feature space. By adjusting the number of neighbors (k), we aimed to find an optimal balance between local and global information for accurate genre classification.

Each method underwent rigorous evaluation using cross-validation to ensure robustness and generalization. Performance was assessed based on mean accuracy scores derived from multiple folds.

Through this extensive exploration, the aim was to identify the most effective approach for music genre classification, considering factors such as accuracy, efficiency, and interpretability.
